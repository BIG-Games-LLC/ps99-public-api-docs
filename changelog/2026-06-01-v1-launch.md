# v1 API

**Released:** 2026-06-01

The new `/v1` namespace is live alongside the existing `/api` surface ‚Äî authenticated
per-user player data, public player profiles, clan and league leaderboards, and a working
OAuth client example.

## What's new

- **OAuth 2.0 + PKCE authentication** for per-user data. Register an app at https://db.biggames.io/settings/developer-apps. See [v1/authentication.md](../v1/authentication.md).
- **Public player profiles** at `/v1/players/:slug`, plus `/search`, `/featured`, a paginated `/v1/players` list, and a `/v1/players/total` count. Players opt in to per-view visibility from their dashboard. See [v1/players.md](../v1/players.md).
- **Seven authenticated `/v1/account/*` endpoints**: inventory, profile, extendedProfile, itemIndex, trades, booth, mail ‚Äî the same 7-view model as the public `/v1/players` surface. See [v1/account.md](../v1/account.md).
- **Aggregated `/v1/clans/*` endpoints**: players aggregator, single-player lookup, battle detail. See [v1/clans.md](../v1/clans.md).
- **`/v1/leagues/*` endpoints**: leaderboard, single-league detail, top contributors, single-contributor lookup. See [v1/leagues.md](../v1/leagues.md).
- **Per-player daily refresh quota** on account endpoints, surfaced in every response. See [v1/refresh-quota.md](../v1/refresh-quota.md).

## Public and account views

`GET /v1/players/:slug?include=` and the `/v1/account/*` endpoints expose seven
privacy-scoped views: `profile`, `inventory`, `extendedProfile`, `itemIndex`, `trades`,
`booth`, and `mail`. Players opt in to each view from their dashboard.

Three views are **pass-through** ‚Äî their `data` is a projection of whitelisted save keys in
the save's native PascalCase shape, with no parser layer reformatting, renaming, or
grouping:

- **`profile`** ‚Äî flat object of whitelisted player-state keys: progression, currencies, statistics, mastery, achievements, zones, and miscellaneous flags.
- **`extendedProfile`** ‚Äî sensitive opt-in, scoped to `Gamepasses`, `Products`, and `RobuxSpent`. A player can publish `profile` while keeping `extendedProfile` private.
- **`itemIndex`** ‚Äî the full raw `ItemIndex` save object (all category buckets).

`inventory`, `trades`, `booth`, and `mail` ship curated envelopes with item-database
enrichment and Roblox username resolution. `inventory` includes the equipped loadout as an
`equipped` field.

See [v1/players.md](../v1/players.md#public-views) and [v1/account.md](../v1/account.md).

### Privacy

The sensitive set (`extendedProfile`) is structurally disjoint from `profile` via the
whitelist ‚Äî there is no parallel field copy, so `profile` never exposes `RobuxSpent`,
playtime, or login data regardless of the `extendedProfile` opt-in.

## Leagues

Leagues are small competitive teams on a continuous points leaderboard ‚Äî similar to
[clans](../v1/clans.md) but simpler (no battles, medals, or diamonds): a name, an icon, an
owner, a roster of up to four, and a running points total.

- **`GET /v1/leagues`** ‚Äî the full leaderboard, paginated and sortable by `Points`, `Members`, `Created`, or `Name`, with an optional `search` filter.
- **`GET /v1/leagues/:name`** ‚Äî single league detail: roster and per-player point contributions, with Roblox display names resolved.
- **`GET /v1/leagues/players`** ‚Äî top point contributors aggregated across all leagues (rolling sample of the top 500 contributions, rebuilt at most once every 3 minutes).
- **`GET /v1/leagues/players/:userId`** ‚Äî single-contributor lookup within that aggregate.

See [v1/leagues.md](../v1/leagues.md).

## Refresh quota

`/v1/account/*` endpoints expose a per-player **daily fresh-data quota**. Every account
response carries a top-level `refresh` object and a set of `X-RateLimit-*` headers
(`Resource: player-refresh`, window 86400s).

- **10 fresh pulls/day** for standard players, **30/day** for VIP ‚Äî counted per Roblox player, per UTC day, shared across the official site and all third-party apps.
- A fresh pull happens only when the cached snapshot is older than 5 minutes **and** the player has logged in since it was taken; otherwise the snapshot is served for free (`refresh.skipped`).
- Exhausting the quota is **not** a 429: the API returns `200` with the last snapshot and `refresh.quotaExhausted: true`.
- Anonymous public reads (`/v1/players/*`) never consume the quota; their `refresh` is `null`.

Full reference: [v1/refresh-quota.md](../v1/refresh-quota.md).

## Item enrichment

Enriched item shapes (inventory items, trade/booth/mail items, equipped pets) include a
**`goldenIcon`** field carrying the pet's golden-variant artwork (empty string for non-pets
and pets without dedicated golden art); render it in place of `icon` for the golden variant.

The equipped-pet shape is
`{ uid, slot, id, displayName, icon, goldenIcon, shiny, rainbow, golden }`. Golden/Rainbow
variants map consistently as `pt === 1` ‚Üí **Golden** and `pt === 2` ‚Üí **Rainbow** across
every endpoint.

## What stayed the same

The existing `/api/*` endpoints are fully supported. See [legacy/README.md](../legacy/README.md).

## Conventions

- Standard response envelope: [v1/overview.md#response-envelope](../v1/overview.md#response-envelope).
- Rate limits for authenticated endpoints: 60/key/min, 120/player/min, 600/app/min.
- `/v1/account/*` responses are `Cache-Control: private, no-cache` ‚Äî never share across users.
- `/v1/players/*` and `/v1/leagues/*` are public, cacheable, and require no authentication.

## Compatibility

`/v1` is pre-GA. Once GA lands, the additive-only compatibility policy in
[v1/overview.md](../v1/overview.md) takes effect.
